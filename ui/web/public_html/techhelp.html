There are four major subsystems:

  1. Modeling User Interface
  2. Compiler
  3. Enactment Mechanism, which we refer to as the PEOS (Process Enactment Operating System); the kernel is further decomposed into:
    1. Event handling mechanism.
    2. Engine, comprising a virtual machine for executing PML models.
    3. Repository, where process state is stored.
  4. Enactment User Interfaces

Modeling UI -- Compiler

The interface between the Modeling UI and Compiler is defined by the PML Grammar, which is part of the PML compiler requirements.

Compiler -- PEOS

The interface between the Compiler and PEOS is defined by the process Virtual Machine language.

Engine -- Repository

The interface between the Engine (virtual machine) and the repository is determined by the needs of the virtual machine to set and query process and product state.

PEOS -- Enactment UI

The interface between PEOS and the various enactment user interfaces is determined by the various Agent interaction scenarios. The following messages are defined:

  1. Create(model) - create a new instance of model
  2. Query(query) - perform a query against the repository.
  3. Query Available Tasks - request a list of tasks in the "ready" state that can be performed by the requesting agent.
  4. Available Tasks - Reply to "Query Available Tasks". A list of actions in the "ready" state that can be performed by the requesting agent.
  5. Select Action(action) - the agent has elected to perform action, which should be marked "active."
  6. Suspend Action(action> - the agent has temporarily stopped performing action; it should be marked "suspended."
  7. Abort Action - the agent has permanently stopped performing action; it should be marked "aborted".
  8. Action Done - the agent has completed action.
  9. Status - reply to request messages that don't explicitly request results.
